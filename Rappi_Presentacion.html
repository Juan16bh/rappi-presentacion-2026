03 · Proceso

Paso a Paso:
Cómo lo construí

PASO 01 · ~10 min

Leer el brief
y explorar el CSV

Leí la prueba completa. Abrí el CSV y analicé: 290 filas, intervalos de 10s, rango 06:11–07:00, valores de 37 a 20,762 tiendas.

Decisión clave

Acotar el análisis a una ventana horaria para hacer el dashboard más legible.

PASO 02 · ~15 min

Diseñar la
arquitectura con Claude

Le pedí a Claude que propusiera una arquitectura React de dos paneles. Juntos definimos los componentes, el flujo de datos y qué modelos usar para cada caso.

Decisión clave

Separar la lógica de datos (availabilityData.js) de los componentes visuales.

PASO 03 · ~20 min

Scaffolding y
componentes base

Claude generó el proyecto Vite+React, la estructura de carpetas y los componentes base. Yo revisé cada archivo y ajusté la lógica de datos reales del CSV.

Decisión clave

Hardcodear el dataset pre-procesado para garantizar la demo sin depender de carga de archivos.

PASO 04 · ~25 min

Dashboard:
KPIs y gráfico

Construí el gráfico principal con Recharts, las 5 KPI cards con skeleton loading, y la línea de SLO de referencia. Iteré 3 veces el diseño de colores.

Decisión clave

Usar AreaChart con fill para visualizar el volumen acumulado de disponibilidad.

PASO 05 · ~20 min

Chatbot:
integrar Claude API

Conecté la Anthropic API directamente desde el browser. El mayor reto fue configurar el header anthropic-dangerous-direct-browser-access correctamente.

Reto encontrado

CORS bloqueaba las requests. Claude me explicó el header especial de Anthropic para acceso directo.

PASO 06 · ~15 min

Narrative Panel
+ SSE Streaming

Agregué el panel de narrativa ejecutiva con Haiku y streaming SSE. El efecto de texto que va apareciendo en tiempo real mejoró mucho la percepción de la IA.

Reto encontrado

Parsear el stream SSE línea a línea sin perder chunks parciales del buffer.

PASO 07 · ~15 min

SRE Framework:
Golden Signals + SLO

Apliqué el framework del Google SRE Book para estructurar los paneles inferiores. La detección de plateau se hizo con análisis estadístico de deltas.

Decisión clave

Usar Golden Signals como marco conceptual demostraba conocimiento de SRE real.

PASO 08 · ~10 min

Polish: SplashScreen,
dark mode, CSV upload

SplashScreen animado con logo de Rappi, toggle dark/light, y cargador de CSV propio para demostrar con datasets alternativos.

Decisión clave

La UX de onboarding (splash + skeleton) hacía la app sentirse más profesional.

NarrativePanel.jsx · SSE Streaming claude-haiku-4-5 
// Leer stream SSE en tiempo real
const reader  = res.body.getReader()
const decoder = new TextDecoder()

while (true) {
  const { done, value } = await reader.read()
  if (done) break
  const chunk = decoder.decode(value, { stream: true })
  for (const line of chunk.split('\n')) {
    if (!line.startsWith('data: ')) continue
    const parsed = JSON.parse(line.slice(6))
    if (parsed.type === 'content_block_delta')
      setText(prev => prev + parsed.delta.text)
  }
}
NarrativePanel.jsx · Detección de plateau Anomaly Detection 
// Detección estadística de plateau en serie
const deltas = sampled.slice(1).map((d, i) =>
  Math.abs(d.value - sampled[i].value)
)
const avgDelta = deltas.reduce(
  (a, b) => a + b, 0
) / (deltas.length || 1)
const flatCount = deltas.filter(
  d => d < avgDelta * 0.1
).length

// Si hay >6 segmentos planos → plateau detectado
const plateauNote = flatCount > 6
  ? `Plateau: ${flatCount} segs < 10% avgDelta`
  : ''
